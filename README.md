# 🎓 StudyMind — AI-Powered Exam Prep Assistant

StudyMind is a full-stack RAG (Retrieval-Augmented Generation) application that lets students upload their study materials and interact with them through an intelligent chat interface and an AI-generated quiz engine. Every user gets their own isolated document collection — nothing is shared between accounts.

## Features

- **Secure Auth** — Sign up / log in via Supabase. JWT tokens protect every API route.
- **Document Upload** — Supports PDF, DOCX, TXT, and Markdown. Files are chunked, embedded, and stored per user.
- **Document Management** — View and delete uploaded files directly from the sidebar.
- **RAG Chat** — Ask questions and get answers grounded in your own documents, with cited source previews and relevance scores.
- **Study Modes** — Q&A, Quiz me, Explain, and Summarise — each applies a different prompt prefix automatically.
- **AI Quiz Generation** — Questions are generated by Groq's LLaMA 4 model using content retrieved from your ChromaDB collection. Supports easy / medium / hard / mixed difficulty, custom topics, configurable question count, and per-question timers.
- **Static Quiz Fallback** — If no documents are uploaded, a built-in question bank is used so the quiz always works.
- **Quick Prompts** — One-tap prompts for common study actions (key topics, exam predictions, summaries, comparisons).
- **Fully Responsive** — Desktop sidebar layout and mobile bottom-nav drawer layout with full feature parity.

## Local Development

**1. Clone the repo and install dependencies:**

```bash
git clone https://github.com/your-username/studymind.git
cd studymind
pip install -r requirements.txt
```

**2. Create your `.env` file** with the variables listed above.

**3. Start the backend:**

```bash
uvicorn main:app --reload --port 8000
```

The API will be available at `http://localhost:8000`. Interactive docs are at `http://localhost:8000/docs`.

**4. Open the frontend:**

Open `index.html` directly in your browser, or serve it with any static file server:

```bash
python -m http.server 3000
# then open http://localhost:3000
```

Make sure `API_BASE` at the top of the `<script>` block in `index.html` points to `http://localhost:8000`.

## Deploying to Render

**1. Prepare your files:**

`requirements.txt`:
```
fastapi
uvicorn[standard]
python-dotenv
colorama
tiktoken
chromadb
groq
supabase
PyJWT
pypdf
python-docx
requests
```

`Procfile`:
```
web: uvicorn main:app --host 0.0.0.0 --port $PORT
```

**2. Push to GitHub** — your repo should contain `main.py`, `requirements.txt`, and `Procfile`.

**3. Create a new Web Service on [Render](https://render.com):**

- Connect your GitHub repo
- **Runtime:** Python 3
- **Build Command:** `pip install -r requirements.txt`
- **Start Command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`

**4. Add Environment Variables** in the Render dashboard — add all six variables from the `.env` section above.

**5. Deploy.** Once live, copy your Render URL (e.g. `https://studymind-api.onrender.com`) and update `API_BASE` in `index.html`:

```js
const API_BASE = "https://studymind-api.onrender.com";
```

Then host `index.html` on any static host (Netlify, Vercel, GitHub Pages, Render Static Site).

## API Reference

All routes except `/health`, `/auth/signup`, and `/auth/login` require a Bearer token in the `Authorization` header.

### Auth

| Method | Route | Description |
|---|---|---|
| `POST` | `/auth/signup` | Register a new account |
| `POST` | `/auth/login` | Log in, receive a JWT token |
| `GET` | `/auth/me` | Return the current user's info |

### Documents

| Method | Route | Description |
|---|---|---|
| `POST` | `/ingest` | Upload and chunk a document (PDF, DOCX, TXT, MD) |
| `GET` | `/collections` | List all documents for the current user |
| `DELETE` | `/documents/{filename}` | Delete all chunks of a specific document |
| `GET` | `/metadata` | Return all unique metadata field values |

### RAG

| Method | Route | Description |
|---|---|---|
| `POST` | `/query` | Ask a question — returns an LLM answer with cited sources |
| `POST` | `/sources` | Retrieve ranked chunks without calling the LLM |
| `DELETE` | `/history` | Clear the conversation history for the current session |

### Quiz

| Method | Route | Description |
|---|---|---|
| `POST` | `/quiz/generate` | Generate MCQ questions from your documents via Groq |

**`POST /quiz/generate` request body:**
```json
{
  "topic": "Photosynthesis",
  "difficulty": "medium",
  "num_q": 5,
  "top_k": 8
}
```

**Response:**
```json
{
  "questions": [
    {
      "q": "What is the primary pigment used in photosynthesis?",
      "opts": ["Chlorophyll", "Carotene", "Xanthophyll", "Melanin"],
      "ans": 0,
      "exp": "Chlorophyll is the main pigment that absorbs light energy for photosynthesis."
    }
  ],
  "source_chunks": 8,
  "topic": "Photosynthesis"
}
```

### Info

| Method | Route | Description |
|---|---|---|
| `GET` | `/stats` | Return chunk count, model info, and document list |
| `GET` | `/health` | Health check — returns `{"status": "healthy"}` |

## Frontend Overview

`index.html` is a fully self-contained single-page app with no build step or framework required.

**Key JavaScript globals:**
- `API_BASE` — backend URL, update this before deploying
- `authToken` — Supabase JWT stored in `localStorage`
- `uploadedDocs` — list of filenames fetched from `/collections` on login

**Key functions:**

| Function | Description |
|---|---|
| `enterApp()` | Called after login — hides auth screen, loads stats and document list |
| `fetchUploadedDocs()` | Calls `GET /collections` and renders the sidebar file list with delete buttons |
| `uploadFile()` | Uploads a file to `/ingest`, then refreshes the document list |
| `deleteDocument(filename, idx)` | Calls `DELETE /documents/{filename}`, then refreshes the list |
| `sendQuestion()` | Posts to `/query` and renders the AI response with source citations |
| `startQuiz()` | Calls `POST /quiz/generate`, falls back to static bank if no docs |
| `renderQuestion()` | Renders the current MCQ with options, timer, and progress bar |
| `showResults()` | Displays the score ring, stats grid, and answer review |

## How the RAG Pipeline Works

1. **Upload** — The file is read by `FileReader` (supports PDF via pypdf, DOCX via python-docx, and plain text).
2. **Chunking** — `TokenChunker` splits the text into 512-token chunks with 64-token overlap using `tiktoken`.
3. **Embedding** — Each chunk is embedded via the HuggingFace Inference API using `all-MiniLM-L6-v2` (384-dimensional vectors).
4. **Storage** — Chunks and embeddings are upserted into the user's isolated ChromaDB collection (named `rag_u_{md5(user_id)[:16]}`). Deduplication is handled by deterministic chunk IDs.
5. **Query** — The user's question is embedded, and the top-K most similar chunks are retrieved by cosine similarity.
6. **Generation** — The retrieved chunks are injected into a system prompt and sent to Groq's LLaMA 4 model. The last 6 conversation turns are included for multi-turn context.

## Quiz Generation

When a user clicks **Start Quiz** with documents loaded:

1. The topic field (or a generic study prompt if blank) is embedded and used to retrieve the top 8 most relevant chunks from ChromaDB.
2. A structured prompt instructs Groq to generate exactly N MCQ questions from that content at the chosen difficulty level, returning a strict JSON array.
3. The response is parsed, validated (4 options, valid `ans` index 0–3), and sent to the frontend quiz engine.
4. If no documents exist or the API is unreachable, the frontend silently falls back to the built-in static question bank so the quiz always works.

Difficulty levels:
- **Easy** — straightforward recall and basic comprehension
- **Medium** — application and moderate reasoning
- **Hard** — deep analysis, edge cases, and advanced reasoning
- **Mixed** — a blend of all three levels
